    Task Overview

    We designed a model that can predict if a tweet is sarcastic or not. A tweet is labeled 1 if it is sarcastic, and 0 if not.



    Referencing F1 Score of Sarcasm Class

    • Minimum:    0.275
    • Threshold:   0.4
    • Peak:           0.605

    Functions


    \[\begin{aligned} Accuracy = \frac{T_P + T_N}{T_P + T_N + F_P + F_N} \end{aligned}\]
    \[\begin{aligned} Precision = \frac{T_P}{T_P + F_P} \end{aligned}\]
    \[\begin{aligned} Recall = \frac{T_P}{T_P + F_N} \end{aligned}\]
    \[\begin{aligned} F_1 = 2 * \frac{Precision * Recall}{Precision + Recall} \end{aligned}\]

    The Training Loop

    The Training loop works by iterating over the training set, and then optimizes based on loss function using a scheduler to speed up performance. Then running a validation set off of the model to determine best model for predicting.

    Training Breakdown



    We've experimented with different number of Epochs, batch sizes and learning rates.


    • Ideal learning rate was between 5e-6 and 1e-6 so 3e-6 was selected.
    • Batch sizes of 15,20,30,40, and 50 were tried, with 20 being the one with the best results.
    • Different starting seeds were tried with no real difference between them.
    • No dataset modifications were considered during training, other than adding a few paraphrased sentences that are automatically labeled as non-sarcastic to provide extra training examples.

    Model Architecture

    Model Architecture

    AttentionWithContext ( )

    The AttentionWithContext class is a second layer, and is based on the model classifications from the Hierarchical Attention Networks for Document Classification paper.

    It is a modified attention that computes a context vector from input hidden state.

    Model Architecture

    TransformerLayer ( )

    This layer uses a pre-trained model to get contextualized embeddings for the input.

    Model Architecture

    ATTClassifier ( )

    It is a classification layer that uses the attention context vector and output of the transformer to make a final prediction.

    Conclusion


    • Model performed over minimum threshold off of a reduced model.
    • The best learning-rate found for this model was 3e-6, with 40 epochs, and batch size of 20 off of XLM.
    • The model would skew towards higher predictions for the negative class than the positive class. (Precision and Recall switched.)

    Complications


    • There were issues gaining access to the CU Research Computing Cluster.
      This made the task of training the model more of a tedious slog than it needed to be.
    • CUDA proved effective, to a point.
      However, some members of the team were unable to use CUDA, due to complications with their specific GPU.
    • Due to CPU restrictions, we were only able to run an XLM model, which under performed compared to an XLML model.
